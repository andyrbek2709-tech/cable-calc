# Руководство по валидации данных

## Проблема
Когда парсер PDF извлекает данные кабельного журнала, он может **пропустить некоторые поля** (особенно мощность, которой часто нет в журналах). Если калькулятор получает нулевые или пустые значения, он всё равно работает, но выдаёт неправильные результаты.

## Решение
Теперь система **требует критические данные** в зависимости от режима:

### Режим "select" (подбор сечения)
**Требует:**
- `power_kw` > 0 (расчётная мощность, кВт) ⚠️ **КРИТИЧЕСКИ**
- `length_m` > 0 (длина линии, м)
- `material` ∈ {Cu, Al}
- `insulation` ∈ {PVC, XLPE}
- `method` ∈ {A1..G} (способ прокладки)

**Ошибка без этого:** ❌ `Ошибка входных данных: power_kw обязателен...`

### Режим "check" (проверка сечения)
**Требует:**
- `section_mm2` > 0 (сечение, мм²) ⚠️ **КРИТИЧЕСКИ**
- `power_kw` > 0 (для расчёта тока)
- `length_m` > 0
- Остальное как выше

**Ошибка без этого:** ❌ `Режим 'check': section_mm2 обязателен...`

### Режим "max_load" (макс. нагрузка)
**Требует:**
- `section_mm2` > 0 (сечение, мм²) ⚠️ **КРИТИЧЕСКИ**
- Остальные параметры окружения

**Не требует:** `power_kw` (расчитывается из допустимого тока)

## Рабочий процесс с PDF

1. **Парсинг PDF** → Получаем:
   ```
   cable_id, cable_mark, section_str, section_mm2, 
   phases, length_m, voltage_kv
   ```

2. **Выбор режима:**
   - **Если есть сечение в журнале** → режим "check"
     - Парсер вытянул `section_mm2` ✓
     - Нужна мощность для проверки
   
   - **Если нужно подобрать сечение** → режим "select"
     - Нужна мощность (её **нет в журнале**, надо вводить отдельно!)
   
   - **Если известно сечение, нужна макс. нагрузка** → режим "max_load"
     - Используем `section_mm2` из журнала

3. **Заполнение данных перед расчётом:**
   - Автоматически из парсера: `cable_id, section_mm2, length_m, material, insulation, method`
   - **Вручную ввести**: `power_kw` (если требуется для режима)
   - Остальное: используются значения по умолчанию

## Пример правильного использования

```python
# Шаг 1: Парсим PDF
parsed_rows = parse_pdf("journal.pdf")
# Результат:
# row.section_mm2 = 35.0  ✓
# row.length_m = 150.0    ✓
# row.power_kw = None     ✗ (нет в журнале)

# Шаг 2: Для режима "check" добавляем мощность вручную
cables_to_check = []
for row in parsed_rows:
    cables_to_check.append(CableInputIn(
        cable_id=row.cable_id,
        section_mm2=row.section_mm2,
        length_m=row.length_m,
        power_kw=30.0,  # ← ВРУЧНУЮ (можно запросить у пользователя)
        material="Cu",
        insulation="PVC",
        method="C"
    ))

# Шаг 3: Батч-расчёт
batch = BatchRequestIn(cables=cables_to_check, mode="check")
results = calculate_batch(batch)

# Результат:
# Для каждого кабеля либо ✓ OK, либо ❌ ERROR с понятным объяснением
```

## Что изменилось

| Раньше | Теперь |
|--------|--------|
| power_kw=None → i_calc=0 → OK | power_kw=None → ❌ ERROR |
| section_mm2=0 → i_allow=0 → OK | section_mm2=0 → ❌ ERROR |
| Неизвестная марка → пропуск | Всё равно обрабатывается (марка опциональна) |

## Диагностика

Если видишь ошибки:
- **"power_kw обязателен"** → режим "select" нужна мощность
- **"section_mm2 обязателен"** → режим "check" нужно сечение из журнала
- **"method ... не поддерживается"** → проверить способ прокладки в журнале

Все ошибки теперь видны в `status="ERROR"` с описанием в `hints[]`.
